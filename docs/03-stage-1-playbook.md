# 第一阶段执行手册

## 阶段目标

第一阶段只做一件核心事情：把真实设备通信验证跑通。

## 需要准备的实物

- ODP 电源 1 台
- PSW 电源 1 台
- Windows 开发电脑 1 台
- 交换机 1 台
- 网线 3 根
- 对应电源线
- 如需备用联调，再准备 USB 线和串口线

## 推荐连接方式

优先使用 LAN。

```text
电脑 --网线-- 交换机 --网线-- ODP
                |
                +--网线-- PSW
```

## 推荐 IP 规划

- 电脑：192.168.1.10
- ODP：192.168.1.101
- PSW：192.168.1.102
- 子网掩码：255.255.255.0

## 电脑要安装的软件

- Python 3.11 或 3.12
- VS Code
- NI-VISA
- Git
- Excel
- 厂家 PC 软件

## Python 环境

```powershell
cd .\power-control-host
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e .
```

## 阶段内建议顺序

1. 确认设备铭牌、型号、固件版本
2. 拍摄设备后面板接口照片
3. 完成 LAN 接线和固定 IP 配置
4. 先用厂家软件或 ping 验证设备在线
5. 运行本工程的配置检查
6. 编写并执行 `*IDN?` 验证脚本
7. 再继续设压、设流、开关输出

## 第一批代码任务

- 校验配置是否可加载
- 能创建 ODP 与 PSW 设备对象
- 能选择 VISA / Socket / Serial 传输方式
- 预留 SCPI 命令入口

## 第一阶段结束时应留下的东西

- 现场设备清单
- 通信方式结论
- 可运行的最小验证脚本
- 命令返回样例
- 后续第二阶段的开发输入
